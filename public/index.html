How it works

Open, GitHub-native, and community-first.

Signups flow through GitHub Issues, newsletters are authored in Markdown, and delivery runs through GitHub Actions plus SendGrid.

Subscribe via GitHub

Drop an issue in the repo — the team processes signups and keeps your data safe and transparent.

Markdown-authored briefs

Every issue is written in Markdown, version-controlled, and auditable in the open repository.

Actions + SendGrid delivery

GitHub Actions triggers the send pipeline; SendGrid handles reliable delivery each month.

Live platform data

Stats are pulled fresh from GitHub so each issue reflects the real state of the BLT ecosystem.

Community-curated content

Contributors shape what gets covered — submit highlights, wins, and updates via PR or issue.

Easy to unsubscribe

One click to opt out. No dark patterns, no re-subscribe tricks — just a clean exit if you need it.
